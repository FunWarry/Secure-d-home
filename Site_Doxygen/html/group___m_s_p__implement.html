Detailed Description

Some MSP Micro-controller Support Package code is provided in the BSP source.
The following MSP function and callback are already implemented

UART

  • HAL_UART_MspDeInit and HAL_UART_MspInit
  • HAL_UART_TxCpltCallback
  • USART2_IRQHandler

EXTI

  • F401RE/L476 EXTI15_10_IRQHandler, EXTI9_5_IRQHandler, EXTI4_IRQHandler and EXTI1_IRQHandler
  • L053R8 EXTI4_15_IRQHandler, EXTI0_1_IRQHandler and EXTI4_15_IRQHandler
  • HAL_GPIO_EXTI_Callback

DMA

  • L476 DMA1_Channel6_IRQHandler and DMA1_Channel7_IRQHandler
  • F401 DMA1_Stream5_IRQHandler and DMA1_Stream6_IRQHandler
  • L053 DMA1_Channel4_5_6_7_IRQHandler

Exact EXTI list depends on soldering options and configured DMA list depends on DMA RX/TX configuration options.

If those functions and callbacks are also to be used by application, the code must be removed or adapted to manage multi-instances

Family-specific code is on dedicated files:

  • vl53l0A1-l053msp.c for Nucleo-L053R8
  • vl53l0A1-x4msp.c for Nucleo-F401RE and Nucleo-L476RG
  • uart_trace.c for generic uart handling

MSP Configure expansion board Interrupt i/o

This function configures GPIO i/o and NVIC. See VL53L0A1_GPIO1_MAP for details To enable interrupt handling use XNUCLEO53L0A1_SetIntrStateId()

The input pullup/pulldown is configured by static configuration VL53L0A1_INTR_PIN_PUPD

Warning
Interrupt is disabled and any pending status flushed out
if initial vl53lx sensor interrupt state is important it must be handled externally (soft trigger)
If interrupt vector is shared it is better to configure and enable only one at a time proper shared management is done but yet some isr could be lost or some other fired spuriously.

DevNodevice number see XNUCLEO53L0A1_dev_e

◆ VL53L0A1_EXTI_IOUnconfigure()

void VL53L0A1_EXTI_IOUnconfigure ( int DevNo)

Un-configure sensor interrupt line to mcu

The line is DeInited (put back to input?)

This function disables the related i/o pin and flush line interrupt pending (not vector) this is to permit shared vector to work. To turn off vector use XNUCLEO53L0A1_SetIntrStateId()

Warning
because vector and line disable are not done atomically with isr disable, a last invalid interrupt may be fired.
Parameters
DevNodevice see XNUCLEO53L0A1_dev_e
